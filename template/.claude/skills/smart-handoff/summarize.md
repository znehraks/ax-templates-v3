# Smart HANDOFF - Contextual Summarization

## 자동 요약 생성

### 1. 완료된 작업 → 핵심 성과

**입력**: 상세한 태스크 목록
**출력**: 압축된 성과 요약

```markdown
## 변환 예시

### Before (상세)
- [x] UserService 클래스 생성
- [x] 로그인 메서드 구현
- [x] 회원가입 메서드 구현
- [x] 비밀번호 해시 처리 추가
- [x] 에러 핸들링 추가

### After (압축)
✅ **사용자 인증 시스템 구현 완료**
- UserService: 로그인/회원가입/비밀번호 해시
```

### 2. 파일 변경 → 영향도 분석

```markdown
## 영향도 분석 템플릿

### 고영향 변경
| 파일 | 변경 | 영향 범위 |
|------|------|----------|
| src/auth/UserService.ts | 신규 | 인증 전체 |

### 저영향 변경
- config 파일 업데이트 (3개)
- 테스트 파일 추가 (2개)
```

### 3. 결정 사항 → 구조화된 기록

```markdown
## 결정 기록 형식

### 🔷 [결정 제목]
- **결정**: [최종 선택]
- **이유**: [핵심 근거 1-2문장]
- **영향**: [영향받는 영역]
```

### 4. 다음 단계 → 즉시 실행 가능 형태

```markdown
## 다음 단계 형식

### 즉시 실행 (다음 세션 시작 시)
1. `npm install` - 의존성 설치
2. `src/components/` 디렉토리에서 작업 재개
3. UserProfile 컴포넌트 구현

### 참조 필요
- 아키텍처: `stages/03-planning/outputs/architecture.md`
- 디자인: `stages/04-ui-ux/outputs/design_system.md`
```

## 스테이지별 요약 초점

### 01-brainstorm
- **강조**: 아이디어, 요구사항
- **약화**: 기술적 세부사항

### 06-implementation
- **강조**: 코드 변경, 기술 결정
- **약화**: 아이디어/기획

### 08-qa
- **강조**: 버그, 테스트 결과
- **약화**: 계획 세부사항

## 품질 검증

생성된 요약 검증 기준:
- **명확성**: 모호한 표현 없음
- **완전성**: 핵심 정보 누락 없음
- **실행 가능성**: 다음 단계가 구체적
