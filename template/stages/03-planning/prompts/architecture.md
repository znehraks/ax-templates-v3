# Architecture Design Prompt

## 사용 모델
Gemini

## 프롬프트

```
당신은 시니어 소프트웨어 아키텍트입니다.

다음 요구사항과 기술 리서치를 바탕으로 시스템 아키텍처를 설계해주세요:

---
## 요구사항
{{REQUIREMENTS}}

## 기술 리서치 결과
{{TECH_RESEARCH}}

## 실현 가능성 평가
{{FEASIBILITY}}
---

## 설계 항목

### 1. 시스템 개요
- 시스템 목적
- 주요 기능
- 비기능적 요구사항 대응

### 2. 아키텍처 스타일
- 선택: Monolithic / Microservices / Serverless / Hybrid
- 선택 근거

### 3. 컴포넌트 설계
각 컴포넌트에 대해:
- 이름
- 책임
- 인터페이스
- 의존성

### 4. 데이터 아키텍처
- 데이터 모델 개요
- 데이터 흐름
- 저장소 전략

### 5. 통합 포인트
- 외부 API
- 서드파티 서비스
- 인증/인가

### 6. 다이어그램 (Mermaid)
- 시스템 컨텍스트
- 컨테이너 다이어그램
- 컴포넌트 다이어그램

## 출력 형식
Mermaid 다이어그램을 포함한 마크다운 문서
```

## 기대 출력
`outputs/architecture.md`
